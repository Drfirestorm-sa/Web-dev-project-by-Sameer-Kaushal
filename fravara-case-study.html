Fravara — AI Smart Glasses

Case Study: Designing the Future of Accessible Technology

Project Overview

Fravara is an AI-powered smart-glasses concept designed to help people interact with information naturally in real time. As the founder and CEO, I led the entire product vision, from initial concept through prototyping and UX design.

The Problem

Information access has become increasingly fragmented across multiple devices and platforms. Students, professionals, and people with accessibility needs face several key challenges:

  • Context Switching Fatigue: Users constantly switch between phones, laptops, and notes, losing focus and momentum.
  • Accessibility Barriers: Many devices aren't optimized for people with motor disabilities, visual impairments, or learning differences.
  • Information Overload: Without intelligent filtering, users drown in irrelevant data instead of actionable insights.
  • Social Isolation in Learning: Remote and hybrid education models lack natural, face-to-face interaction cues.
Market Insight

The global AR/VR market is projected to reach $297 billion by 2028, yet accessibility remains severely underdeveloped.

The Solution

Fravara is a wearable AI system that acts as a natural, intelligent interface to the world around you.

AI-Powered NLP

Natural language processing understands user intent and context, delivering relevant information through voice commands or gestures.

  • Real-time transcription
  • Intent recognition
  • Context awareness
Accessibility First Design

Built from the ground up with accessibility at the core, ensuring inclusivity for all users.

  • Hands-free navigation
  • Voice-only mode (no screen reading needed)
  • Customizable UI density
Learning Enhancement

Supports active learning through real-time visual annotations, summarization, and spaced repetition prompts.

  • Text-to-speech explanations
  • Visual highlighting & annotations
  • Quiz mode integration
Privacy by Design

On-device processing and encrypted data handling ensure user privacy and data security.

  • Local processing (no cloud required)
  • User-controlled data
  • GDPR compliant

Design Process

Phase 1

User Research & Ideation

Conducted interviews with students, educators, and accessibility advocates. Identified key pain points and brainstormed 15+ prototypes using design thinking methodology.

Phase 2

Lo-Fi Wireframing

Created low-fidelity wireframes of the HUD (Heads-Up Display) showing key interactions. Tested with 8 users from diverse backgrounds.

Phase 3

Hi-Fi Prototyping

Built a functional prototype in JavaScript with a Python backend. Integrated real-time NLP processing and gesture recognition (C++).

Phase 4

Usability Testing

Conducted 5 rounds of iterative usability testing with accessibility-focused user groups. Achieved 95% task completion rate in final round.

Lessons Learned

Accessibility Requires Iteration

Our first prototype was too visually complex. After testing with users, we learned that simplicity and voice-first design are non-negotiable for true accessibility.

User Input is Gold

Talking to real users revealed insights no amount of theorizing could provide. A wheelchair-using student suggested the gesture recognition feature that became one of our key differentiators.

Privacy Concerns Matter

Users were initially hesitant about the cameras/sensors in glasses. Implementing on-device processing and clear privacy controls dramatically increased trust and adoption intent.
