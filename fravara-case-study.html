Fravara — AI Smart Glasses

Case Study: Designing the Future of Accessible Technology

  1. Home
  2. Projects

Project Overview

Fravara is an AI-powered smart-glasses concept designed to help people interact with information naturally in real time — built with accessibility and learning at its core.

As founder and CEO, I led every aspect: product vision, system architecture, UX research, and prototype development.

Role
Founder & CEO, Lead Designer
Timeline
2025 — Present
Tech Stack
Python, JavaScript, C++, NLP
Team
Solo founder, mentor collaboration

The Problem

People face barriers that existing devices don't solve well:

  • Context Switching Fatigue — constantly jumping between phones, laptops, and notes.
  • Accessibility Barriers — most devices aren't built for users with motor or visual impairments.
  • Information Overload — no intelligent filtering means users drown in irrelevant data.
  • Learning Disconnection — remote education lacks natural interaction cues and real-time support.

Market Insight

The global AR/VR market is projected to reach $297 billion by 2028, yet accessibility features remain severely underdeveloped.

The Solution

Fravara is a wearable AI system that acts as a natural, intelligent interface to the world around you.

AI-Powered NLP

Real-time natural language understanding via voice and gesture commands.

  • Real-time transcription
  • Intent recognition
  • Context awareness
Accessibility First

Designed from the ground up for inclusive, hands-free interaction.

  • Hands-free navigation
  • Voice-only mode
  • Customisable UI density
Learning Enhancement

Real-time visual annotations, summarisation, and spaced repetition prompts.

  • Text-to-speech explanations
  • Visual highlighting
  • Quiz mode integration
Privacy by Design

On-device processing and encrypted handling protect every user.

  • Local processing (no cloud required)
  • User-controlled data
  • GDPR compliant

Design Process

Phase 1

User Research & Ideation

Conducted interviews with students, educators, and accessibility advocates. Identified key pain points and brainstormed 15+ prototype directions using design thinking methodology.

Phase 2

Lo-Fi Wireframing

Created low-fidelity wireframes of the HUD (Heads-Up Display) showing key interactions. Tested with 8 users from diverse backgrounds and iterated rapidly.

Phase 3

Hi-Fi Prototyping

Built a functional prototype with a JavaScript frontend and Python NLP backend. Integrated real-time gesture recognition (C++ / OpenCV).

Phase 4

Usability Testing

Ran 5 rounds of iterative usability testing with accessibility-focused user groups, achieving a 95% task completion rate in the final round.

Technical Implementation

Backend Architecture

  • NLP Engine: Python (spaCy, NLTK)
  • Voice Processing: Python (librosa, SpeechRecognition)
  • Data Storage: SQLite — local, optional cloud sync
  • API Integration: REST API for external services

Frontend & Hardware

  • UI Framework: JavaScript (Vue.js)
  • Gesture Recognition: C++ (OpenCV)
  • Display: AR-enabled lens simulation
  • Microcontroller: Arduino-compatible prototype

Lessons Learned

Accessibility Requires Iteration

Our first prototype was too visually complex. Testing revealed that simplicity and voice-first design are non-negotiable for true accessibility.

User Input is Gold

A wheelchair-using student suggested the gesture recognition feature that became one of our core differentiators — no theorising would have found that.

Privacy Concerns Matter

Users were hesitant about wearable cameras. On-device processing and clear privacy controls dramatically increased trust and adoption intent.
